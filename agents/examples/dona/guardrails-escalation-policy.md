> מתי, איך ולמי מסלימים. משלים את [`../orchestration/escalation.md`](../orchestration/escalation.md) (מנגנון ה‑payload)
> ואת [`../orchestration/supervisor.md`](../orchestration/supervisor.md) (שיפוט ההסלמה).

## עקרון‑על

הסלמה היא **שירות**, לא כישלון, ולעולם לא דד‑אנד. כל הסלמה מגיעה מסודרת (סיכום, הקשר, דחיפות, צעד הבא) ליעד
האנושי הנכון, והמשתמש מקבל אישור שקוף.

## טבלת טריגרים → יעד → דחיפות

| טריגר | סוכן | יעד אנושי | דחיפות |
|---|---|---|---|
| הצפה/קצר/גז/סדק מבני/סכנה לאדם | בדק בית | `*2966` / שירות דיירים | **P1** |
| ליקוי בתחום אחריות שדורש טכנאי | בדק בית | `*2966` / שירות דיירים | P2/P3 |
| "פניתי כבר כמה פעמים" / לקוח כועס | כל סוכן | Supervisor → שירות דיירים | P2 |
| דרישת פיצוי / זיכוי / מחווה | כל סוכן | Supervisor → נציג דונה | P2 |
| בקשת ייעוץ מחייב / חתימה / סכסוך / איום בתביעה | משפטי | Supervisor → עו"ד | P2 |
| מחלוקת על אחריות / פרשנות סעיף שנויה במחלוקת | משפטי/בדק | עו"ד / שירות דיירים | P2 |
| אירוע/סיכון בטיחות באתר | קבלנים | מנהל עבודה / מהנדס | **P1** |
| חריגה מהיתר / סתירת תכנון / החלטה הנדסית | קבלנים | מנהל פרויקט / מהנדס | P2 |
| אין מידע מאומת / מחוץ לתחום | כל סוכן | הגורם המתאים לתחום | לפי העניין |

## חוקי זהב

1. **בטיחות וכסף לא מוכרעים ע"י סוכן** - תמיד Supervisor → אדם.
2. **לא מבטיחים פיצוי/תוצאה** - רק "אעביר לגורם המוסמך שיבחן".
3. **שקיפות:** אומרים למי הועבר, מה צורף, ומה הצעד/הזמן הצפוי (⚑ זמני יעד לאימות מול דונה).
4. **הסכמה לשיתוף PII** לפני העברת פרטים אישיים.
5. **תיעוד:** כל הסלמה → Escalation history + Observability (audit trail).
