# Orchestrator - שכבת תזמור ובקרה (Layer 4, "המוח")

> ה‑Orchestrator הוא הרכיב המרכזי שעוטף את שלושת הסוכנים ומחבר בין כל רכיבי השכבה. הסוכנים הם הכוכבים;
> השכבה הזו היא הרקמה המחברת שגורמת להם להתנהג כאנשי מקצוע, לא כתפריט.

## תפקיד

1. לקבל הודעה מערוץ (WhatsApp / צ'אט אתר).
2. להפעיל **Intent & Routing** → לזהות מי המשתמש ומה הכוונה → לבחור סוכן.
3. להעביר לסוכן המתאים עם ההקשר (זיכרון רלוונטי, שפה).
4. לנטר (Observability) ולאכוף Guardrails ו‑Compliance לאורך כל השיחה.
5. כשהמצב מורכב/רגיש/בעל סיכון - להעביר ל‑**Supervisor** (ולא לסוכן הבא).

## ארכיטקטורה ב‑5 שכבות (מה שבנינו בפועל - ללא המצאות)

הארכיטקטורה בנויה ב‑5 שכבות. כך **מה שבנינו ל‑Dona** יושב בכל שכבה - ובמכוון **רק רכיבים אמיתיים ורלוונטיים**;
מה שלא רלוונטי לשלושת הסוכנים סומן במפורש כ"מחוץ לתחום" (זו אותה משמעת אנטי‑הזיה שאנו דורשים מהסוכנים).

| שכבה | מה שבנינו ל‑Dona | מה **לא** כללנו (ולמה) |
|---|---|---|
| **01 תזמור (Orchestrator)** | Intent & Routing + תיעדוף לפי דחיפות + ניהול הקשר + **Supervisor** (תוספת ערך שלנו) | - |
| **02 סוכנים ייעודיים** | שלושת הסוכנים: **בדק בית · משפטי · קבלנים** (תפורים לדונה) | לא "Sales/Support" גנריים - אלה התחומים של דונה |
| **03 זיכרון ושכבת נתונים** | מודל 6 שכבות זיכרון (vector + episodic), namespace לכל סוכן, סנכרון פרויקט | - |
| **04 שכבת אינטגרציות** | CRM/Ticketing (פתיחת קריאה), WhatsApp, Booking (זימון ביקור) | **תשלומים - מחוץ לתחום בכוונה**; **ניקוד לידים - לא רלוונטי** לשלושת הסוכנים |
| **05 ממשל ו‑Observability** | Observability (audit, bot‑vs‑human, דחיפות) + Compliance (PII/RBAC/הצפנה) + הרשאות | - |

> **הערה (אנטי‑הזיה על המוצר עצמו):** גם בעמוד התצוגה מציגים **רק** רכיבים שתוכננו כאן בפועל. אין ניקוד לידים,
> תשלומים, או מחברים שלא נבנו - בדיוק כפי שהסוכנים אינם ממציאים מידע.

## מחזור חיים של בקשה (Request lifecycle)

```
ערוץ → [Language: זיהוי שפה] → [Intent & Routing: תפקיד + כוונה]
     → [Compliance/Guardrails: סינון קלט] → סוכן מומחה
        ├─ מענה מעוגן בידע (RAG, relevant‑only) → [Guardrails: סינון פלט] → תשובה למשתמש
        └─ טריגר הסלמה/סיכון/חוסר‑ביטחון → [Supervisor] → אדם (עם payload מלא)
     → [Observability: תיעוד כל החלטה/הסלמה/תוצאה]
```

## עקרונות תזמור

- **ניתוב זול וקבוע מול פיקוח יקר וחכם:** ה‑Router מהיר וזול (סיווג כוונה). ה‑Supervisor הוא ה"מנהל" -
  נכנס רק כשיש סיכון, כסף (פיצוי) או מורכבות. שום פעולה בלתי‑הפיכה או רגישה לא עוקפת אותו.
- **סוכן בודד בכל רגע:** משתמש מדבר עם סוכן אחד; מעבר בין סוכנים נעשה ב‑handoff מסודר דרך ה‑Orchestrator
  (לא קפיצה שקטה), עם שמירת הקשר.
- **Fail‑safe:** כשל בכלי/בידע לעולם לא מוביל להמצאה - מוביל ל"אין לי מידע מאומת" + הסלמה.
- **אג'נוסטי למודל:** הפלטפורמה מקצה את המודל; ההיגיון כאן אינו תלוי‑ספק.
