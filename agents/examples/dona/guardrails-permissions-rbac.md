# היררכיית הרשאות (Permissions / RBAC)

> מנגנון Role‑Based Access Control (RBAC). קובע מי רואה מה, מי מורשה לאיזו פעולה, ואיפה נדרש אישור אנושי.
> נאכף ע"י ה‑Orchestrator (Intent & Routing) ושכבת ה‑Compliance.

## תפקידים והרשאות

| תפקיד | רואה | מורשה לפעולות | חסום |
|---|---|---|---|
| **אורח / לא מזוהה** | מידע כללי בלבד | שאלות כלליות | כל פעולת כתיבה, כל מידע אישי |
| **דייר / רוכש** | הדירה/הפרויקט **שלו בלבד** | לשאול, לקבל מידע, **לפתוח קריאת שירות**, לבקש טיוטה, לזמן ביקור | מידע של דיירים אחרים, עריכת Vault, אישור פיצוי |
| **קבלן / איש ביצוע** | הפרויקט/ים **שהוקצו לו** | מושגים, נהלים, **בקשת תיאום עבודה**, חיפוש מונחים | פרטי דיירים, החלטות סמכות/תכנון, עריכת Vault |
| **נציג דונה / אדמין** | חוצה‑פרויקטים (פנימי) | הכול + **עריכת Vault** + **אישור הסלמה/פיצוי** | - (כפוף ל‑audit trail) |

## זיהוי הפונה (תיאור הפונה) - תת‑הרשאות בקריאת שירות

מתוך טופס השירות של דונה, הפונה מסווג ל‑**בעלים · שוכר · ועד בית · חברת ניהול · מיופה כוח**. הסיווג קובע מה הסוכן יכול לספק:

| תיאור הפונה | היקף מותר |
|---|---|
| **בעלים** | מלא לדירתו: פתיחת קריאה, מעקב, טיוטות, בקשות |
| **מיופה כוח** | כמו בעלים - **בכפוף לאימות ייפוי כוח** |
| **שוכר** | דיווח תקלות בדירה שהוא שוכר; נושאי בעלות/חוזה/שינויים מהותיים → אישור הבעלים |
| **ועד בית** | תקלות ב**רכוש המשותף/שטחים ציבוריים** בלבד; לא בדירות פרטיות |
| **חברת ניהול** | ניהול הבניין/רכוש משותף לפי הרשאתה |

עיקרון: הסוכן **מזהה קודם מי פנה** ומתאים את היקף השירות. אם הפעולה חורגת מהרשאת הפונה - מסביר, ומפנה/מסלים לבעל ההרשאה.

## עקרונות

1. **בידוד נתונים:** כל שליפת זיכרון/פרויקט מסוננת לפי המשתמש - אין דליפה בין דיירים/פרויקטים.
2. **זיהוי לפני כתיבה:** פעולת כתיבה (קריאת שירות, תיאום, טיוטה) דורשת משתמש מזוהה + **אישור מפורש** שלו.
3. **שערי אישור אנושי (human approval gates):**
   - עריכת Legal Doc Vault → אדמין/עו"ד.
   - אישור **פיצוי/מחווה** → נציג דונה/אדמין (דרך Supervisor).
   - פעולות בעלות השלכה כספית/משפטית/בטיחותית → אדם.
4. **הרשאות כלים נגזרות מהתפקיד:** ראו מיפוי בכלים → [`../tool_calling/tool_logic.md`](../tool_calling/tool_logic.md).

## דוגמה

דייר מבקש "תראה לי את הליקויים של השכן" → **נחסם** (בידוד נתונים) + הסבר אדיב.
קבלן מבקש פרטי דייר → **נחסם** (אין הרשאה ל‑PII של דיירים) + הפניה לערוץ המתאים.
