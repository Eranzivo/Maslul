# Intent & Routing - זיהוי כוונה וניתוב

> רכיב סיווג כוונה (intent classifier) + לוגיקת הניתוב המרכזית.
> תפקיד: לקבוע **מי המשתמש** + **מה הוא רוצה** → לבחור את הסוכן הנכון, או להסלים.

## שלב 1 - זיהוי תפקיד (Role)

| תפקיד | סימנים מזהים | דיפולט הרשאה |
|---|---|---|
| **דייר / רוכש** | "קניתי דירה", "קיבלתי מפתח", "יש לי רטיבות", "סעיף בחוזה" | גישה לנתוני הדירה/פרויקט שלו בלבד |
| **קבלן / איש ביצוע** | "אני קבלן המשנה", "אישור יציקה", "תיאום עבודה באתר" | גישה לפרויקט/ים שהוקצו לו |
| **נציג דונה / אדמין** | התחברות מאומתת פנימית | חוצה‑פרויקטים + עריכת Vault + אישור הסלמות/פיצוי |
| **לא מזוהה / אורח** | אין הקשר | מענה כללי בלבד; פעולות כתיבה חסומות עד זיהוי |

ראו מודל ההרשאות המלא: [`../guardrails/permissions.md`](../guardrails/permissions.md).

## שלב 2 - סיווג כוונה (Intent) → סוכן

תוויות הכוונה (classifier מרובה‑תוויות). כל תווית ממופה לסוכן:

| כוונה | סוכן יעד |
|---|---|
| ליקוי/תקלה אחרי מסירה, רטיבות, סדק, אחריות, פתיחת קריאת שירות, זימון בדיקה | **בדק בית** |
| הבנת סעיף בחוזה, מסמך מסירה, טיוטת מסמך, מושג מקרקעין, ערבות/טאבו | **משפטי וחוזים** |
| מושג בנייה, תיאום עבודות, נוהל/תהליך ביצוע, טופס 4/היתר, שאלת שטח | **קבלנים וביצוע** |
| תלונה מורכבת, דרישת פיצוי, איום משפטי, סכסוך, מקרה דחוף/בטיחותי | **→ Supervisor** (לא לסוכן) |
| מחוץ לתחום שלושת הסוכנים (מכירות, תשלומים, משאבי אנוש) | מענה כללי + הפניה למוקד דונה |

## שלב 3 - כללי ניתוב

- **כוונה כפולה:** אם ההודעה נוגעת לשני תחומים (למשל "יש לי רטיבות וגם לא הבנתי סעיף אחריות בחוזה") -
  לטפל קודם בדחוף (בדק בית/בטיחות), ואז להציע handoff מסודר לסוכן השני.
- **ביטחון נמוך בסיווג:** אם ה‑classifier לא בטוח (מתחת לסף) - לשאול **שאלת הבהרה אחת** במקום לנחש.
- **טריגרי סיכון עוקפים ניתוב רגיל:** מילות דחיפות/בטיחות ("הצפה", "קצר חשמלי", "נפילת אריחים", "סכנה")
  או רגש קיצוני/איום → ישירות ל‑Supervisor עם דחיפות P1.
- **מעבר בין סוכנים** מתבצע תמיד דרך ה‑Orchestrator עם העברת הקשר (לא "השלכה" של המשתמש להתחיל מחדש).
