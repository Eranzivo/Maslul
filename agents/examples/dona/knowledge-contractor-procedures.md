# בסיס ידע - נהלים ומושגי ביצוע (קבלנים ושטח)

> מקור האמת של **סוכן הקבלנים**. מושגים ונהלים כלליים בענף הבנייה בישראל. כל קביעה נשענת על מקור;
> החלטות סמכות/תכנון/בטיחות מוסלמות לאדם (מנהל פרויקט/מהנדס).

## 1. רצף רגולטורי (מהיתר עד אכלוס)

```
היתר בנייה → ביצוע (לפי תוכניות מאושרות) → טופס 4 (אישור אכלוס) → תעודת גמר (טופס 5)
```

- **היתר בנייה:** אישור הוועדה המקומית לבנות לפי תוכניות.
- **טופס 4 (אישור אכלוס):** תנאי **לחיבור חשמל/מים/גז ולאכלוס** המבנה; מאשר שאין סכנה ושניתן להשתמש למטרה
  המוצהרת.
- **תעודת גמר (טופס 5):** אישור גמר הבנייה הרשמי; לרוב לאחר טופס 4.

## 2. מילון מושגי ביצוע (לתשובות מהירות)

| מושג | פירוש קצר |
|---|---|
| **מפרט טכני** | תיאור מחייב של חומרים/גימורים/מערכות בדירה |
| **כתב כמויות** | פירוט סעיפי עבודה וכמויות לתמחור ומעקב ביצוע |
| **יומן עבודה** | תיעוד יומי של התקדמות, כוח אדם, אירועים באתר |
| **גמר חשבון** | חשבון סופי מול קבלן/קבלן משנה בסיום עבודה |
| **פיקוח עליון** | פיקוח המתכנן/מהנדס שהביצוע תואם לתכנון |
| **מנהל עבודה** | אחראי ניהול האתר והבטיחות בפועל |
| **בדיקות מעבדה** | בדיקות בטון/חומרים לעמידה בתקן |
| **אישורי יועצים/קונסטרוקטור** | אישורים נדרשים לשלבי ביצוע/אכלוס |
| **נהלי בטיחות באתר** | כללי בטיחות מחייבים; חריגה = עצירה והסלמה |

## 3. תיאום עבודות (Coordination)

- בקשת תיאום כוללת: פרויקט, סוג עבודה, מועד מבוקש, תלות בעבודות אחרות, דרישות בטיחות/אישורים.
- הסוכן מסייע **לארגן ולהבהיר** את הבקשה ולתאם מול לוח הזמנים - אך **אישור סופי** (חריגה, שינוי תכנון,
  סמכות) הוא של מנהל הפרויקט/המהנדס.
- כלי `schedule_coordination` יוצר בקשת תיאום מסודרת (write → אישור לפני שליחה).

## 3א. שינויי דיירים (תיאום ביצוע)

הרוכש רשאי לשנות את דירתו מהסטנדרט; הביצוע מתואם דרך סוכן הקבלנים:
- **סוגי שינויים:** במפרט (ריצוף/כלים/ברזים), בתוכניות (נקודות חשמל/מים), ותכנוניים (מיקום קירות, מספר חדרים).
- **מועד קריטי:** יש להגיש ולסגור שינויים **לפני יציקת הדירה**, כדי שצוות הביצוע יממש אותם נכון.
- **טפסים נפוצים:** הסכמה על תוכניות סטנדרט, זיהוי ריצופים/כלים, השלמת בחירות ספקים, סגירת תיק דייר.
- **מסחרי:** מחירון שינויים + זיכויים בגין עבודות שלא בוצעו. ⚑ מחירון/זיכויים ספציפיים של דונה - מול החברה.
- הצד המשפטי/מסמכי (טופס בקשת שינויים) נמצא ב‑Legal Doc Vault (`tmpl.tenant_change_request`).

## 4. מתי הסוכן עוצר ומסלים (ראו `guardrails/escalation_policy.md`)

- **בטיחות:** אירוע/סיכון באתר → עצירה מיידית + הסלמה למנהל עבודה/מהנדס (P1).
- **תכנון/סמכות:** סתירה בתוכניות, חריגה מהיתר, החלטה הנדסית → מהנדס/מנהל פרויקט.
- **מחוץ לידע:** אם אין נוהל/מקור מאומת → "אין לי מידע מאומת" + הסלמה. לא מאלתרים נהלים.

מקורות: ויקיפדיה (טופס אישור אכלוס), נתיבי הקמה, עיריית ת"א (איכות הבנייה), ארכאונליין + CivilEng (שינויי דיירים). ראו `docs/research_notes.md`.
⚑ נהלים פנימיים ספציפיים של דונה - להשלים מול החברה (לא הומצאו כאן).
